# راه‌اندازی اندروید

اپ اندروید پروکسی رله را مستقیماً روی گوشی اجرا می‌کند — بدون نیاز به دسکتاپ.

## پیش از شروع

ابتدا راه‌اندازی رله را کامل کن:
- [Apps Script دیپلوی شده باشد](../../README_FA.md#مرحله-۲--راهاندازی-apps-script)
- [رله خروجی در حال اجرا باشد](../../README_FA.md#مرحله-۳--راهاندازی-رله-خروجی) (VPS یا Cloudflare)

## نصب اپ

APK را از صفحه [Releases](../../../releases) دانلود کن و روی گوشی نصب کن.

اگر اندروید نصب را مسدود کرد: **Settings → Install unknown apps** → به مدیر فایل اجازه بده.

## راه‌اندازی اولیه

### ۱. وارد کردن کانفیگ

ساده‌ترین راه از طریق برنامه دسکتاپ است:

1. در برنامه دسکتاپ، روی دکمه **export** (دانلود) در بخش Tools کلیک کن
2. JSON نمایش داده شده را کپی کن
3. در گوشی، **Import Config from Clipboard** را بزن

کانفیگ به طور خودکار ذخیره می‌شود. می‌توانی چندین کانفیگ از راه‌اندازی‌های مختلف وارد کنی و بین آن‌ها جابجا شوی.

**اگر برنامه دسکتاپ نداری**، JSON را دستی بساز:

<div dir="ltr">

```json
{"url":"https://script.google.com/macros/s/ID_شما/exec","key":"کلید_امنیتی"}
```

</div>

برای چند آدرس Apps Script (پایداری بهتر):

<div dir="ltr">

```json
{"url":"https://script.google.com/.../exec1,https://script.google.com/.../exec2","key":"کلید_امنیتی"}
```

</div>

### ۲. نصب گواهینامه CA

برای سایت‌های HTTPS لازم است. اگر فقط سایت‌های HTTP نیاز داری، این مرحله را رد کن.

1. در اپ روی **Install CA Certificate** بزن
2. گواهینامه در `Downloads/zyrln-ca.pem` ذخیره می‌شود
3. وقتی پرسیده شد، **Open Settings** را بزن
4. برو به **Biometrics & security → Other security settings → Install from device storage**
   - در اندروید stock: **Settings → Security → Encryption & credentials → Install a certificate → CA certificate**
5. از پوشه Downloads فایل `zyrln-ca.pem` را انتخاب کن
6. وقتی پرسیده شد نوع را **CA certificate** انتخاب کن

> ⚠️ هرگز فایل `.pem` را از کامپیوتر به گوشی کپی نکن. هر دستگاه CA منحصربه‌فرد خودش را می‌سازد. استفاده از گواهینامه اشتباه باعث خطای SSL می‌شود. همیشه از دکمه **Install CA Certificate** داخل اپ استفاده کن.

### ۳. اتصال

1. روی یک کانفیگ در لیست بزن تا انتخاب شود (هایلایت می‌شود)
2. دکمه **connect** (آیکون پاور، بالا راست) را بزن
3. وقتی اندروید درخواست VPN داد، اجازه بده
4. دکمه سبز شد — متصل شدی

برای قطع: دوباره دکمه connect را بزن.

## حالت مستقیم (سرویس‌های گوگل)

دکمه **⚡ برق** حالت مستقیم را فعال می‌کند — سرویس‌های گوگل (یوتیوب، جیمیل، درایو) با TLS تکه‌تکه شده مستقیم به گوگل متصل می‌شوند، نه از طریق رله. سریع‌تر است و سهمیه کمتری مصرف می‌کند.

با زدن دکمه برق فعال می‌شود (سبز می‌شود). مستقل از رله عمل می‌کند — می‌توانی هر دو را همزمان فعال داشته باشی.

## پینگ

**ping** را بزن تا زمان رفت و برگشت رله را اندازه بگیری. از کانفیگ انتخاب‌شده یا فعال استفاده می‌کند. برای مقایسه چند راه‌اندازی رله مفید است.

## ساخت APK از سورس

نیاز به Android Studio با NDK دارد.

<div dir="ltr">

```bash
# نصب gomobile (یک بار)
go install golang.org/x/mobile/cmd/gomobile@latest
gomobile init

# ساخت
make keystore        # ساخت کلید امضا (یک بار)
make android         # APK نهایی امضاشده

# نسخه دیباگ (بدون keystore)
make android-debug
```

</div>

مسیر APK: `android/app/build/outputs/apk/release/`

## عیب‌یابی

**خطای SSL در سایت‌های HTTPS**
گواهینامه CA نصب نشده یا مورد اعتماد نیست. مرحله ۲ را دوباره انجام بده. در گوشی‌های سامسونگ ممکن است مسیر زیر **Biometrics & security → Other security settings** باشد.

**بعضی اپ‌ها از طریق پروکسی کار نمی‌کنند**
اپ‌هایی که گواهینامه TLS خودشان را دارند (اپ‌های بانکی، بعضی اپ‌های پرداخت) پروکسی سیستم را نادیده می‌گیرند. بدون دسترسی root نمی‌توان این را برطرف کرد.

**دسترسی VPN رد شد**
برو به **Settings → Apps → Zyrln → Permissions** و دسترسی VPN را دستی بده.

**کانفیگ وصل نمی‌شود**
ابزار **Diagnostics** دسکتاپ را اجرا کن تا مطمئن شوی زنجیره رله کار می‌کند، قبل از اینکه به اندروید منتقل کنی.
