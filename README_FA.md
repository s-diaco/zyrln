# زیرلن (Zyrln)

یک رله (Relay) مبتنی بر تکنیک Domain-Fronting که ترافیک شما را از زیرساخت‌های گوگل عبور می‌دهد تا سیستم‌های سانسور مبتنی بر DPI را دور بزند.

## نحوه عملکرد

```
دستگاه شما
  ← پروکسی محلی (Go)
  ← اسکریپت گوگل (Apps Script)   ← برای سیستم سانسور شبیه ترافیک عادی گوگل به نظر می‌رسد
  ← رله خروجی (VPS)
  ← سایت مقصد
```

اتصالات TLS به محدوده‌های آی‌پی (IP) گوگل برقرار می‌شوند. هدر `Host` که رمزنگاری شده است، اسکریپت شما در گوگل را هدف قرار می‌دهد. از دید سیستم فیلترینگ (DPI)، این ترافیک با ترافیک معمولی گوگل (مانند جستجو یا درایو) غیرقابل تشخیص است.

## ویژگی‌ها

- **غیرقابل شناسایی توسط DPI** — تمام ترافیک از آی‌پی‌های گوگل خارج می‌شود. هیچ اثر انگشت VPN، پورت غیرمعمول یا آی‌پی سرور اختصاصی برای مسدود کردن وجود ندارد.
- **تجمیع درخواست‌ها (Request coalescing)** — درخواست‌های همزمان مرورگر در یک فراخوانی اسکریپت تجمیع می‌شوند. این کار باعث می‌شود به جای ۳۰ فراخوانی، تنها ۱ تا ۳ بار از سهمیه روزانه گوگل استفاده شود.
- **کش داخلی (In-proxy response cache)** — فایل‌های ثابت (JS، CSS، فونت‌ها و تصاویر) در مراجعات بعدی از حافظه محلی سرو می‌شوند که سرعت بارگذاری را به شدت افزایش می‌دهد.
- **Failover موازی چندلینکی** — امکان پیکربندی چندین اسکریپت گوگل روی اکانت‌های مختلف. زیرلن لینک‌های تنظیم‌شده را موازی امتحان می‌کند و اولین پاسخ موفق را استفاده می‌کند تا اگر یک لینک کند یا بی‌سهمیه شد، کاربر معطل نشود.
- **پشتیبانی کامل از HTTPS** — پروکسی عملیات رمزگشایی و رمزنگاری را به صورت محلی انجام می‌دهد تا سایت‌های HTTPS فیلتر شده به درستی کار کنند. هیچ داده غیررمزنگاری شده‌ای از دستگاه خارج نمی‌شود.
- **اندروید بدون نیاز به روت** — با یک لمس، تمام ترافیک مرورگر از طریق رله عبور می‌کند. نیازی به تنظیمات پیچیده یا دسترسی روت نیست.
- **پیکربندی‌های ذخیره شده** — در نسخه اندروید می‌توانید بی‌نهایت کانفیگ ذخیره کنید و به راحتی بین آن‌ها جابه‌جا شوید.
- **ابزار بررسی دسترسی (Probe)** — ابزار خط فرمان دسکتاپ می‌تواند قبل از راه‌اندازی، تمام اجزای شبکه را تست کند تا دقیقاً بدانید چه بخش‌هایی در شبکه شما به درستی کار می‌کنند.

## قبل از شروع

**فایل‌های اجرایی و APK به تنهایی کار نمی‌کنند.** زیرلن یک کلاینت است و رله داخلی ندارد. شما باید ابتدا زنجیره رله خود را راه‌اندازی کنید:

| پیش‌نیاز | دلیل |
|---|---|
| اکانت گوگل | برای اجرای رایگان اسکریپت رله |
| سرور مجازی (VPS) | برای اجرای رله خروجی (یا استفاده از Cloudflare Worker) |
| کلید احراز هویت (Auth key) | یک رمز مشترک که تمام اجزا را به هم متصل می‌کند |

یک کلید اختصاصی برای خود بسازید و آن را ذخیره کنید:

```bash
openssl rand -base64 32
# مثال: swrkwbMS1X666fjzReip+PbodKcPyDK7Xbk5gRSgRUE=
```

## راه‌اندازی

### ۱. راه‌اندازی اسکریپت گوگل (Apps Script)

این درب ورودی است — ترافیک را دریافت کرده و به سرور شما می‌فرستد.

۱. به [script.google.com](https://script.google.com) بروید و یک پروژه جدید بسازید.
۲. محتویات فایل `relay/apps-script/Code.gs` را کپی کرده و در آنجا قرار دهید.
۳. مقادیر ثابت را در بالای فایل تنظیم کنید:

```js
const AUTH_KEY       = "YOUR_KEY";           // کلیدی که در مرحله قبل ساختید
const EXIT_RELAY_URL = "http://YOUR_VPS_IP:8787/relay";
const EXIT_RELAY_KEY = "";                   // فعلاً خالی بگذارید
```

۴. بر روی **Deploy ← New deployment ← Web app** کلیک کنید.
   - Execute as: **Me**
   - Who has access: **Anyone**
۵. لینک دیپلوی را کپی کنید. چیزی شبیه به این است:
   `https://script.google.com/macros/s/AKfycb.../exec`

> **سهمیه:** هر اکانت گوگل ۲۰,۰۰۰ درخواست در روز اجازه می‌دهد. می‌توانید چندین لینک را با کاما (,) در تنظیمات خود وارد کنید. زیرلن لینک‌ها را موازی امتحان می‌کند؛ پس چندلینکی بودن بیشتر برای پایداری و کاهش تأخیر هنگام خرابی یا اتمام سهمیه کاربرد دارد.

### ۲. راه‌اندازی رله خروجی (VPS یا Cloudflare)

این گره خروجی است که درخواست‌های واقعی را از اینترنت دریافت می‌کند. یکی از دو روش زیر را انتخاب کنید:

**گزینه الف — سرور مجازی (VPS):**
فایل **[docs/fa/vps-setup.md](docs/fa/vps-setup.md)** را برای نحوه نصب و اجرا مطالعه کنید.

**گزینه ب — Cloudflare Worker (رایگان):**
فایل `relay/cloudflare/worker.js` را به عنوان یک Worker دیپلوی کنید. راهنما: **[docs/fa/cloudflare-setup.md](docs/fa/cloudflare-setup.md)**.

### ۳. تنظیم پروکسی دسکتاپ

در نسخه آماده دسکتاپ، رابط گرافیکی داخل خود فایل اجرایی قرار دارد و لازم نیست پوشه‌های سورس کنار برنامه باشند.

فایل `config.env` را بسازید:

```
fronted-appscript-url = https://script.google.com/macros/s/YOUR_ID/exec
auth-key              = YOUR_KEY
listen                = 127.0.0.1:8085
socks-listen          = 127.0.0.1:1080
```

کلید `listen` برای پروکسی محلی HTTP/HTTPS است.
کلید `socks-listen` برای شنونده محلی SOCKS5 است.
اگر `socks-listen` را در تنظیمات GUI وارد نکنید، مقدار پیش‌فرض `127.0.0.1:1080` استفاده می‌شود.

برای چند لینک اسکریپت (بدون فاصله):
```
fronted-appscript-url = link1,link2,link3
```

اگر از سورس اجرا می‌کنید، ابتدا برنامه را بسازید. سپس گواهینامه امنیتی (CA) را ایجاد کنید:
```bash
make desktop && ./zyrln -init-ca
```

فایل `certs/zyrln-ca.pem` را به عنوان یک Authority در مرورگر خود Import کنید.

اجرای پروکسی:

- **ویندوز**: فایل `run.bat` را کنار `zyrln-windows-amd64.exe` بگذارید و روی `run.bat` دو بار کلیک کنید.
- **لینوکس/مک**: دستور `make gui` را اجرا کنید.

سپس یکی از این دو حالت را در کلاینت خود تنظیم کنید:

- پروکسی HTTP: مقدار HTTP و HTTPS مرورگر را روی `127.0.0.1:8085` بگذارید.
- پروکسی SOCKS5: آدرس SOCKS را روی `127.0.0.1` و پورت را روی `1080` یا مقدار `socks-listen` خود بگذارید.

## اپلیکیشن اندروید

فایل **[docs/fa/android-setup.md](docs/fa/android-setup.md)** را برای راهنمای کامل نصب و اجرا مطالعه کنید.

۱. فایل APK را نصب کنید.

## نکات امنیتی

- هر کاربر باید اسکریپت گوگل و کلید اختصاصی خود را داشته باشد.
- هرگز فایل `config.env` یا کلیدهای خود را در جایی منتشر نکنید.
- گواهینامه امنیتی محلی (`certs/zyrln-ca-key.pem`) را با کسی به اشتراک نگذارید.

## محدودیت‌ها

- **فقط مرورگر:** این یک پروکسی HTTP است، نه یک VPN کامل. فقط ترافیک مرورگر و اپلیکیشن‌هایی که از پروکسی سیستم پیروی می‌کنند عبور داده می‌شود. اپلیکیشن‌هایی مثل اینستاگرام یا تلگرام مستقیماً متصل می‌شوند (مگر اینکه پروکسی آن‌ها را دستی تنظیم کنید).
- **سهمیه گوگل:** ۲۰ هزار درخواست در روز برای سایت‌های سنگین ممکن است زود تمام شود.
- **حجم فایل:** درخواست‌های بالای ۱۲ مگابایت به دلیل محدودیت گوگل ممکن است با خطا مواجه شوند.

## اشتباهات رایج (Common Mistakes)

⚠️ **کپی کردن گواهینامه (Certificate) از کامپیوتر به گوشی:** این یکی از رایج‌ترین اشتباهات است. هر دستگاه (ویندوز، مک، اندروید) برای خود یک گواهینامه اختصاصی می‌سازد. شما نباید گواهینامه ویندوز را به گوشی منتقل کنید. در اپلیکیشن اندروید حتماً از دکمه **Install CA Certificate** خود برنامه استفاده کنید.
